# DocPage2MD 使用说明

这是 DocPage2MD 的 Windows GUI 发布包。请先解压整个文件夹，再运行：

```text
DocPage2MD\DocPage2MD.exe
```

不要只把 `DocPage2MD.exe` 单独拷走运行。这个程序依赖同目录下的 `_internal` 文件夹。

## 1. 第一次使用

1. 打开 `DocPage2MD.exe`。
2. 在“模型/Provider”或“设置”区域配置 API Key。
3. 推荐先用“检查 Key”确认本机能读到 Key，再用“验证模型”确认模型可用。
4. 回到主界面添加 PDF，选择工作流，设置输出目录，然后点击开始。

API Key 可以保存到本机环境变量、本地 ignored secrets 文件或 Windows Credential Manager。程序日志、命令预览和报告不会写入真实 Key。

## 2. 推荐工作流

- 只想快速转 Markdown：选择 MinerU 或 PaddleOCR 单引擎。
- 想要更高质量：选择 MinerU + PaddleOCR 双引擎融合。
- 需要 AI 修正文档：选择带“精修”的混合模式。
- PDF 很长时：程序会按页数自动分段，再合并为一个最终文档。

默认建议使用：

- MinerU 模型版本：`vlm`
- PaddleOCR 模型：`PaddleOCR-VL-1.6`
- 输出保留模式：标准或精简
- Brain 上下文窗口：前后 2 页

## 3. 输入文件

主界面可以添加：

- 单个 PDF
- 多个 PDF
- 文件夹
- 已有 MinerU/PaddleOCR artifact
- URL

文件表格会显示文件名、类型、大小、页数和限制状态。页码范围可以留空，表示处理全部页面。

## 4. 输出文件

输出目录中常见文件包括：

- `Slide_XX.md`：每页 Markdown
- `*_FULL.md`：合并后的完整 Markdown
- `assets/`：Markdown 引用的图片资源
- `run_report.json`：任务报告
- `process.log`：运行日志

精简模式会减少调试文件，适合日常使用。调试或完整审计模式会保存更多原始证据，占用更多空间。

## 5. 常见问题

### 双击没有反应

请不要只运行单独的 exe。确认 `DocPage2MD.exe` 和 `_internal` 文件夹在同一个 `DocPage2MD` 目录里。

### API Key 检查失败

先确认 Key 保存位置是否正确。可以在 GUI 里重新保存，也可以使用环境变量。

### 验证模型失败

这通常是网络、余额、模型名或 Base URL 问题。先检查 Provider 设置，再看 `process.log`。

### 输出目录很大

把输出保留模式改成“精简”。调试和完整审计模式会保存更多原始响应、图片和审计信息。

### 扫描 PDF 页数不准

当前版本使用 `pypdf` 优先读取页数，失败时回退轻量估算。若页数仍不准，请先指定页码范围再运行。

## 6. 命令行版本检查

如果需要确认版本，可以在命令行运行：

```powershell
DocPage2MD.exe --docpage2md-cli --version
```

当前发布版本：`0.1.0`
