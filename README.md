# PPT2MD-Qwen-VL 🚀

> 基于 **Qwen3-VL-Plus** (通义千问视觉大模型) 的智能 PPT 转 Markdown 工具。
> 支持 **公式识别 (LaTeX)**、**图表深度理解**、**上下文连贯** 以及 **多任务并发**。

## 📖 简介

本项目旨在将大量的课件/论文/会议 PPT 图片批量转换为结构清晰的 Markdown 笔记。

不同于普通的 OCR 工具，本项目利用了大模型的**视觉推理能力**：

1. **精准公式**：自动识别行内/行间公式并转换为标准 LaTeX 格式 (`$$...$$`)。
2. **上下文连贯**：模型“记得”上一页讲了什么，避免重复输出，同时保证知识点的连续性（通过 `<CTX>` 隐藏状态块实现）。
3. **视觉描述**：自动分析架构图、流程图、数据图表，并生成详细的文字描述。
4. **智能断点续传**：任务中断后，下次运行自动检测进度，无需重跑已完成的页面。

## ✨ 核心特性 (V5.0)

* **📂 自动分组与隔离**：支持按文件夹管理不同的 PPT 任务，输出自动分类。
* **⚡ 多进程并发**：支持同时处理多个 PPT（如 `MAX_WORKERS=3`），每个 PPT 内部保持串行以维护上下文。
* **🧠 深度思考模式**：启用 `Thinking Mode`，极大提升对复杂图表的理解能力（控制台隐藏思考过程，只展示进度）。
* **🎮 交互式控制**：启动时自动扫描任务，支持选择特定 PPT、指定页码范围（如 `1-10`, `5-end`）。
* **⏱️ 智能计时**：实时显示已用时间、平均速度及预计剩余时间 (ETA)。
* **🔄 断点续传**：自动跳过已生成的文件，并读取前一页的上下文状态，无缝接续生成。
* **📑 自动合并**：单页生成完毕后，自动汇总为 `XXX_FULL.md`。

1. **Python 环境**
需要 Python 3.8 或更高版本。
2. **安装依赖**
本项目依赖阿里云官方 SDK。
```bash
pip install -U dashscope

```


3. **配置 API Key**
你需要开通阿里云百炼服务并获取 `Qwen3-VL-Plus` 的权限。
*方式 A (推荐 - 环境变量)*:
```powershell
# Windows PowerShell
$env:DASHSCOPE_API_KEY="sk-你的密钥"

# Linux / macOS
export DASHSCOPE_API_KEY="sk-你的密钥"

```


*方式 B (修改代码)*:
打开 `ppt2md.py`，修改第 12 行：
```python
dashscope.api_key = "sk-你的密钥"

```

## 🚀 使用指南

### 1. 启动程序

在终端运行：

```bash
python ppt2md.py

```

### 2. 交互式选择

程序启动后会扫描 `ppt_images` 下的所有子文件夹，并进入交互模式：

```text
🔄 正在扫描 PPT 文件夹...

========================================
📂 发现以下 PPT:
  [1] 课件_原子物理01 (80 页)
  [2] 课件_原子物理02 (20 页)

👉 请选择要处理的 PPT:
   输入序号(如 1,3) | 输入 'a' 全选
   您的选择: 

```

### 3. 配置页码范围

选择 PPT 后，你可以指定处理范围（支持断点续传逻辑）：

* `all` 或 `回车`：处理所有页面。
* `1-10`：只处理第 1 到第 10 页。
* `20-end`：从第 20 页处理到最后。

### 4. 运行过程

程序将启动多进程池（默认 3 个并发），你将看到实时进度：

```text
[课件_原子物理01] 启动! 共 80 页
[课件_原子物理02] 启动! 共 20 页

[课件_原子物理01] 处理 P1/80: slide_01.png
  >> 状态: 正在深度思考...
  >> 生成: # Slide 1 ... (流式输出内容)
  
[课件_原子物理01] 处理 P2/80: slide_02.png | 耗时: 0:00:45 | 均速: 45.0s/页 | 剩: 0:58:30

```

## ⚙️ 高级配置

在 `ppt2md.py` 顶部可以调整以下参数：

```python

MODEL_NAME = "qwen3-vl-plus"

# 并发数：同时处理几个文件夹？
# 注意：Qwen API 有 RPM 限制，建议保持在 3-5 之间
MAX_WORKERS = 3  

# 思考预算：控制模型推理的深度 (Token数)
THINKING_BUDGET = 2048 

```

## ❓ 常见问题 (FAQ)

**Q: 为什么运行到一半报错 `list index out of range` 或 `KeyError`?**
A: V5.0 版本已修复此问题。这是由于流式传输中的空包导致的，新代码增加了 `safe_get_content` 保护机制。

**Q: 如果任务中断了，会覆盖已生成的文件吗？**
A: **不会**。脚本会检测 `output` 目录下是否存在非空的目标文件。如果存在，会打印 `跳过 (断点续传)` 并自动读取该文件的上下文，直接开始下一页。如果想重跑某页，请手动删除对应的 `.md` 文件。

**Q: 输出的 Markdown 最后有 `<CTX>` 标签，我不想要怎么办？**
A: 在代码 `merge_markdowns` 函数中，取消 `re.sub` 行的注释即可在合并时自动过滤掉这些元数据。

**Q: 为什么不能所有图片同时并发？**
A: 为了保证 PPT 的**上下文连贯性**（比如第 2 页引用第 1 页的定理），同一个 PPT 内必须按顺序（串行）处理。但不同的 PPT 之间是完全并发的。
